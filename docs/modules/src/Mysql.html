Mysql¶

Usage

use Mysql;
proc MysqlConnectionFactory(host: string, user: string = "", database: string = "", passwd: string = ""): MysqlConnection¶

Creates a connection to the MySQL server and returns a MysqlConnection object

Arguments:
  • host : string – The host name or IP address of the MySQL server.
  • user : string – The user name used to authenticate with the MySQL server.
  • database : string – The database name to use when connecting with the MySQL server.
  • passwd : string – The password to authenticate the user with the MySQL server.
class MysqlConnection: ConnectionBase¶
override proc cursor()¶

This method returns a MysqlCursor object. The returned object is a CursorBase instance.

override proc Begin()¶

This method starts a transaction. You must commit after every transaction that modifies data for tables that use transactional storage engines.

override proc commit()¶

This method sends a COMMIT statement to the MySQL server, committing the current transaction. By default Connector/Chapel does autocommit, but call this method after every transaction that modifies data for tables that use transactional storage engines.

override proc rollback()¶

This method sends a ROLLBACK statement to the MySQL server, undoing all data changes from the current transaction.

override proc setAutocommit(commit: bool)¶

This method can be used to assign a value of True or False to enable or disable the autocommit feature of MySQL.

arg commit:Value to enable or disable the autocommit.
type commit:bool
proc isAutoCommit(): bool¶

This method can be called to retrieve the current autocommit setting.

override proc close()¶

This method tries to send a QUIT command and close the socket. It raises no exceptions.

class MysqlCursor: CursorBase¶
proc rowcount(): int(32)¶

This method returns the number of rows returned for SELECT statements, or the number of rows affected by DML statements such as INSERT or UPDATE.

override proc close()¶

This method closes the cursor, resets all results, and ensures that the cursor object has no reference to its original connection object.
